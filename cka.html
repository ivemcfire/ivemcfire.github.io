Lab setup — the "3-2-1" you type cold at the start of every lab

You cannot access this page during the exam (PSI proctored session locks the browser to kubernetes.io). This block has to come out of your fingers without thinking. Mental shortcut: 3 lines + 2 lines + 1 line. Three groups, six lines total.

Group A — make k work like kubectl (3 lines)

alias k=kubectl
source <(kubectl completion bash)
complete -F __start_kubectl k

Why three? Line 1 makes k exist. Line 2 loads kubectl's tab-completion functions into the shell. Line 3 binds those completion functions to the k alias. Without line 3, only kubectl <TAB> autocompletes — not k <TAB>. That single line saves hundreds of keystrokes per exam.

Group B — two exports for speed (2 lines)

export do='--dry-run=client -o yaml'
export now='--grace-period=0 --force'

Memory hooks:

  • $do = "do as if" (rehearsal, no real action). Use: k run web --image=nginx $do > web.yaml — writes the YAML, doesn't create the resource.
  • $now = "do it NOW" (no 30-second grace period, kill it instantly). Use: k delete pod stuck $now — gone immediately. Saves a half-minute every stuck-pod scenario.

Group C — vim safe for YAML (1 line)

printf 'set ts=2 sw=2 et\nset pastetoggle=<F2>\nset nu\n' > ~/.vimrc

Three vim settings packed into one file:

  • ts=2 sw=2 et — tab-stop and shift-width = 2, expand tabs to spaces. YAML breaks on literal tabs — this prevents the most common silent-fail when editing manifests.
  • pastetoggle=<F2> — hit F2 in vim before pasting YAML from docs, hit again to resume editing. Without it, vim auto-indents the paste and shreds the structure.
  • nu — line numbers. Easier to jump with :42 when something errors at "line 23 col 5."

Verify everything took (one-liner)

echo "$do" && type k && k get no

Should print the $do string, "k is aliased to kubectl", and the node list. Then sanity-check autocomplete: type k get po + TAB — should expand to pods. If it doesn't, line 3 of Group A didn't run.

The whole block, no commentary (copy-paste reference)

alias k=kubectl
source <(kubectl completion bash)
complete -F __start_kubectl k
export do='--dry-run=client -o yaml'
export now='--grace-period=0 --force'
printf 'set ts=2 sw=2 et\nset pastetoggle=<F2>\nset nu\n' > ~/.vimrc

Recall drill (use during prep)

Test yourself: close your eyes, can you say "3 lines, 2 lines, 1 line" and what each group does?

  • 3 = k works: alias, source completion, bind completion to k.
  • 2 = $do, $now: dry-run-yaml-redirect, force-grace-zero.
  • 1 = vim YAML: ts/sw/et + pastetoggle + nu, all into ~/.vimrc.

Debug pods — one-shot pods for in-cluster testing

Spin a one-shot pod to test from INSIDE the cluster. Most Services/Networking and DNS questions need this — ClusterIP services are not reachable from the bastion/control-plane node.

One-shot patterns (all use --rm -it --restart=Never)

# Shell into busybox
k run tmp --image=busybox --rm -it --restart=Never -- sh

# HTTP test (wget is built into busybox)
k run tmp --image=busybox --rm -it --restart=Never -- wget -O- web-svc:80

# DNS lookup — USE busybox:1.28 (latest's nslookup is broken)
k run tmp --image=busybox:1.28 --rm -it --restart=Never -- nslookup web-svc

# curl (needs a richer image)
k run tmp --image=nicolaka/netshoot --rm -it --restart=Never -- curl web-svc:80

# Long-running debug pod (stay open, exec in later)
k run debug --image=busybox --restart=Never -- sleep 3600
k exec -it debug -- sh

Flag breakdown

FlagMeaning
--rmdelete pod when it exits (no stale pods between drills)
-itinteractive + TTY (output streams to your terminal)
--restart=Nevermakes it a Pod, not a Deployment
--END of kubectl args. Everything after = command run inside the pod. Without it, kubectl errors on "unknown flag."
-O- (wget)output to stdout — the - is the filename. Capital O, not zero.

Gotchas

  • -- is not optional. Forgetting it = "unknown flag" error.
  • -O- = capital-O-dash. -0- (zero) silently fails.
  • busybox:latest has a broken nslookup — use busybox:1.28 for DNS.
  • Service DNS: <svc> works in same namespace. Cross-namespace: <svc>.<ns> or <svc>.<ns>.svc.cluster.local.
  • Without --rm, the pod stays Completed and clutters k get pods for the rest of the question.

In-pod commands (BusyBox)

hostname           # pod name
hostname -i        # pod IP
nslookup <svc>     # CoreDNS check (busybox:1.28)
wget -O- <svc>:<port>
nc -zv <svc> <port>
cat /etc/resolv.conf
