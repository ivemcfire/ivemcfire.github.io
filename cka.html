CKA cheat sheet

Study-phase reference. Not usable during the actual exam — PSI proctored session is locked to kubernetes.io.

Lab setup — almost nothing to do

alias k=kubectl + completion are PRE-SET in both KodeKloud and the real CKA (PSI) exam. Do NOT waste clock typing them. Old guides that tell you to are outdated.

Optional exports (yours, not pre-set)

export do='--dry-run=client -o yaml'
export now='--grace-period=0 --force'

Lost on every ssh node01 / new terminal — re-paste after a context switch, or just type the full flags. Fine to skip entirely.

Verify (k is already there)

type k && k get po          # k get po + TAB expands to pods

$nowforce-grace-zero · k delete pod x $now

Editor: nano (default choice)

Nano handles YAML correctly out of the box. No setup needed. Just nano file.yaml, edit, Ctrl-O save, Ctrl-X exit.

Editor choice: see the Vim tab. k edit defaults to vim, so survival vim is the safer bet.

Paste, don't type (kills exact-name fails)

Never transcribe a string the grader checks. Select it and paste it.
  • Paste the nouns: resource names, namespaces, image tags, ports, paths — straight from the question text (it's selectable). No typo = no silent zero.
  • Paste docs YAML skeletons, then paste the question's names into them.
  • Always paste: base64 secrets, tokens, join commands, long flags.
  • Type the verbs: short imperative commands (k get, k create, k describe) — faster typed, and a typo just errors instead of silently failing.

Maps to memorize-vs-lookup: verbs typed, nouns pasted. Terminal paste = Ctrl+Shift+V.

Don't let a pasted command auto-run

Docs "copy" buttons and triple-click grab the trailing newline — that newline is Enter, so a pasted command fires before you can read it.
  • Single-line command: type # first, then paste. The whole line is commented — it prints, doesn't run. Read it, up-arrow, strip the #, run. (# beats echo: no $/quote/backtick expansion.)
  • Multi-line / YAML: # only protects the first line. Paste into the editor instead (vim :set paste first).

No IDE in the exam — yq is there though

The post-2025 remote-desktop CKA is terminal + vim/nano + a Firefox docs window. No VS Code, no code-server, no IDE. Don't plan around one.

yq (the YAML jq) is pre-installed and underused — the fast path for a one-field read/edit without opening an editor:

yq '.spec.replicas' deploy.yaml             # read a field
yq -i '.spec.replicas = 3' deploy.yaml      # edit in place
yq '.spec.containers[0].image' pod.yaml      # drill into arrays

Workloads — create / inspect / modify

Create

# Pod (imperative)
k run NAME --image=IMAGE [--restart=Never] [--port=P] [-- CMD]

# Deployment
k create deployment NAME --image=IMAGE [--replicas=N] [--port=P]

# Generate YAML only, don't create (uses $do)
k run NAME --image=IMAGE $do > pod.yaml
k create deployment NAME --image=IMAGE $do > deploy.yaml

Expose — deployment/pod → service

k expose deployment NAME --port=P [--target-port=Q] [--type=T] [--name=X]
k expose pod NAME --port=P [--name=X]
FlagDefaultNotes
--portrequiredport the SERVICE listens on
--target-port= --portport the CONTAINER listens on
--typeClusterIPNodePort | LoadBalancer | ClusterIP
--name= deployment nameoverride service name

Inspect

k get pods [-A] [-n NS] [-o wide|yaml|json|jsonpath=...] [-l KEY=VAL]
k describe pod NAME [-n NS]
k logs POD [-n NS] [-f] [--previous] [-c CONTAINER] [--tail=N] [--since=10m]
k exec -it POD [-n NS] [-c CONTAINER] -- CMD

Modify in place

k scale deployment NAME --replicas=N
k set image deployment/NAME CONTAINER=NEW_IMAGE
k set image deployment/NAME --all=NEW_IMAGE     # every container
k edit deployment NAME                          # opens $EDITOR
k label pod NAME key=value [--overwrite]
k label pod NAME key-                           # remove label

Rollout

k rollout status deployment/NAME      # wait until complete
k rollout history deployment/NAME     # list revisions
k rollout undo deployment/NAME        # revert to previous
k rollout undo deployment/NAME --to-revision=2
k rollout restart deployment/NAME     # force pod re-creation

Delete

k delete pod NAME [$now]              # $now = grace=0 --force
k delete deployment NAME              # also kills RS + pods
k delete -f file.yaml
k delete pod -l key=value             # by label selector

Gotchas

  • Container name = image basename AT CREATE TIME (incl. typos: --image=ngnix → container ngnix). It's the CONTAINER token for set image. Don't assume it — look it up: k describe deploy/NAME | grep -i image.
  • k set image deployment/NAME container=image uses =, not space.
  • k rollout status blocks until rollout completes (or stalls). If it hangs, image is failing — check describe pod.
  • k scale changes replica count immediately; no rollout, no new revision.
  • k delete deployment cascades to ReplicaSet + pods automatically. k delete pod on a deployment-owned pod just re-creates it.

Scheduling — Labels, Taints/Tolerations, nodeSelector

Labels & selectors — query

k get po -l env=dev                   # equality filter
k get po -l env=prod,bu=finance       # comma = AND (no spaces)
k get po -l 'env in (dev,prod)'       # set-based
k get po -l env -l '!tier'            # key exists / key absent
k get all -l env=prod                 # ALL object types, not just pods
k get po --show-labels                # every label, one column
k get po -L env -L tier               # labels as their own columns
k get po --no-headers | wc -l         # count (or subtract the header)

Labels — set / change / remove

k label po web tier=frontend          # add
k label po web env=prod --overwrite   # change existing (needs flag)
k label po web tier-                  # remove (trailing dash)
k label nodes node01 size=Large       # label a node (for nodeSelector)
Selector must match labels or it silently breaks: a Service with the wrong spec.selector = 0 endpoints; a ReplicaSet/Deployment whose selector.matchLabels ≠ template labels is rejected at create (`selector` does not match template `labels`). The selector is immutable after create — fix the template to match it.

Taints vs nodeSelector — opposite directions

MechanismDirectionSet on
Taint + Tolerationrepels (keep off unless tolerated)node taint + pod toleration
nodeSelector / affinityattracts (put me here)node label + pod field

Toleration only permits, never attracts — a tolerating pod may still land elsewhere. "Only this pod, only on this node" = taint the node (repel others) + nodeSelector (pull this pod). One alone isn't enough.

Taints (imperative)

k taint nodes node01 spray=mortein:NoSchedule        # key=value:effect
k describe node node01 | grep -i taint               # check
k taint nodes node01 spray=mortein:NoSchedule-       # remove (trailing dash)
EffectMeaning
NoScheduleblock new pods without the toleration
PreferNoSchedulesoft — avoid if possible
NoExecuteblock + evict already-running non-tolerating pods

Control-plane ships with node-role.kubernetes.io/control-plane:NoSchedule — that's why workloads avoid it.

Toleration / nodeSelector (pod YAML, under spec:)

  tolerations:
  - key: "spray"
    operator: "Equal"        # Exists = any value (omit value:)
    value: "mortein"
    effect: "NoSchedule"
  nodeSelector:
    size: Large              # exact equality ONLY (no OR/NotIn)

Node affinity (when nodeSelector can't — OR / NotIn / Exists)

  affinity:
    nodeAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:   # hard; preferred... = soft
        nodeSelectorTerms:
        - matchExpressions:
          - { key: size, operator: In, values: [Large, Medium] }

Operators: In · NotIn · Exists · DoesNotExist · Gt · Lt.

Gotchas

  • Toleration value is string-exact — a typo (mortain ≠ mortein) leaves the pod Pending: the "but I added a toleration!" trap. Paste the noun.
  • No matching node (nodeSelector / required affinity) = pod Pending forever.
  • Pending diagnosis: k describe po X → Events → "untolerated taint" / "didn't match node selector".
  • comma = AND in selectors; get all excludes cm/secret/pvc/ingress/netpol.

Docs path

  • Labels = memorize. k8s.io search "taints and tolerations" · "assign pods to nodes" (affinity).

Debug pods — one-shot pods for in-cluster testing

Spin a one-shot pod to test from INSIDE the cluster. Most Services/Networking and DNS questions need this — ClusterIP services are not reachable from the bastion/control-plane node.

One-shot patterns (all use --rm -it --restart=Never)

# Shell into busybox — ALWAYS pin :1.28 (latest's wget AND nslookup lie)
k run tmp --image=busybox:1.28 --rm -it --restart=Never -- sh

# HTTP test (wget is built into busybox)
k run tmp --image=busybox:1.28 --rm -it --restart=Never -- wget -O- web-svc:80

# DNS lookup — USE busybox:1.28 (latest's nslookup is broken)
k run tmp --image=busybox:1.28 --rm -it --restart=Never -- nslookup web-svc

# curl (needs a richer image)
k run tmp --image=nicolaka/netshoot --rm -it --restart=Never -- curl web-svc:80

# Long-running debug pod (stay open, exec in later)
k run debug --image=busybox --restart=Never -- sleep 3600
k exec -it debug -- sh

Flag breakdown

FlagMeaning
--rmdelete pod when it exits (no stale pods between drills)
-itinteractive + TTY (output streams to your terminal)
--restart=Nevermakes it a Pod, not a Deployment
--END of kubectl args. Everything after = command run inside the pod. Without it, kubectl errors on "unknown flag."
-O- (wget)output to stdout — the - is the filename. Capital O, not zero.

Gotchas

  • -- is not optional. Forgetting it = "unknown flag" error.
  • -O- = capital-O-dash. -0- (zero) silently fails.
  • busybox:latest lies — broken nslookup AND a wget that returns a false connection refused. Pin busybox:1.28 for all probes.
  • connection refused ≠ blocked. Refused/RST = traffic reached the host, nothing listening (or a reject-CNI policy). timeout = packets dropped (drop-CNI policy / no route). Different diagnoses.
  • Running ≠ serving. STATUS=Running only means the container's main process is alive. A -- sleep override → Running but the app never started → port closed. Confirm with a probe, not the phase.
  • Service DNS: <svc> works in same namespace. Cross-namespace: <svc>.<ns> or <svc>.<ns>.svc.cluster.local. Bare pods get NO DNS — target them by IP.
  • Without --rm, the pod stays Completed and clutters k get pods for the rest of the question.

In-pod commands (BusyBox)

hostname           # pod name
hostname -i        # pod IP
nslookup <svc>     # CoreDNS check (busybox:1.28)
wget -O- <svc>:<port>
nc -zv <svc> <port>
cat /etc/resolv.conf

Docs path

  • kubernetes.io/docs/tasks/debug/debug-application/debug-running-pod/
  • Search: "debug pod"

Storage & Config — PV / PVC, ConfigMap, Secret

PV + PVC + Pod (static bind, hostPath)

apiVersion: v1
kind: PersistentVolume
metadata: { name: pv-log }
spec:
  capacity: { storage: 1Gi }
  accessModes: ["ReadWriteOnce"]
  persistentVolumeReclaimPolicy: Retain
  storageClassName: manual
  hostPath: { path: /mnt/data }
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata: { name: pvc-log }
spec:
  accessModes: ["ReadWriteOnce"]
  resources: { requests: { storage: 1Gi } }
  storageClassName: manual
---
# in pod.spec:
  volumes:
  - name: data
    persistentVolumeClaim: { claimName: pvc-log }
  # container.volumeMounts:
  - name: data          # MUST == volumes[].name
    mountPath: /usr/share/nginx/html

accessModes & binding

ModeMeaning
RWOReadWriteOnce — one node, read-write
ROXReadOnlyMany — many nodes, read-only
RWXReadWriteMany — many nodes, read-write
RWOPReadWriteOncePod — one pod only
  • PVC binds only if: storageClassName equal · modes ⊆ · PV capacity ≥ request.
  • Static bind (no provisioner): set storageClassName: manual (or "") on BOTH.
  • reclaim: Retain · Delete (dynamic default) · Recycle (dead).

ConfigMap / Secret (imperative)

k create configmap app-cfg --from-literal=KEY=val --from-literal=K2=v2
k create configmap app-cfg --from-file=config.txt
k create secret generic db-sec --from-literal=password=s3cret
# consume: envFrom:[configMapRef|secretRef] | valueFrom.configMapKeyRef | volume

Gotchas

  • volume name must == volumeMount name — mismatch = silent no-mount, no error.
  • PVC stuck Pending = no PV matches (class/access/size). k describe pvc tells you why.
  • emptyDir: {} = scratch (dies with pod). Can't shrink a PVC. RWO ≠ multi-node.
  • Secret values are base64 — --from-literal encodes for you.

Docs path

Networking — NetworkPolicy

The model (the whole game)

SlotMeans
spec.podSelector (top)WHO you protect — the destination. {} = all pods in ns.
ingress.from[].podSelectorWHO is allowed in — the source.

Read the task — "traffic TO x FROM y" — straight into the two slots. The two selectors are different pods: read each one's real label off the question, don't autopilot app:. A wrong key on the top selector matches nothing → the policy applies to no pod → silent allow-all. policyTypes listed with no matching rules = deny-all that direction (Egress alone cuts DNS).

Allow ingress pod → pod (the 90% case)

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata: { name: allow-web-to-db, namespace: prod }
spec:
  podSelector:
    matchLabels: { app: db }       # protected (TO)
  policyTypes: [Ingress]
  ingress:
  - from:
    - podSelector:
        matchLabels: { app: web }  # allowed (FROM)
    ports:                         # optional; omit = all ports
    - { protocol: TCP, port: 80 }

Variants — swap the from block

  • From a namespace: - namespaceSelector: {matchLabels: {kubernetes.io/metadata.name: dev}}
  • ns AND pod: both selectors in one - item · ns OR pod: two separate - items.
  • Deny-all ingress: podSelector: {} + policyTypes: [Ingress] + no ingress rules.
  • Egress: policyTypes: [Egress] + egress: [{to: [...], ports: [...]}].

Verify both directions (always) — busybox:1.28

# destination IP
k get pod DST -n ns -o wide
# allowed: prober carries the trusted LABEL -> serves the page
k run t --image=busybox:1.28 --labels=<allowed-label> -n ns --rm -it --restart=Never -- wget -qO- --timeout=2 <ip>:80
# blocked: no label -> blocked (policy working)
k run t --image=busybox:1.28 -n ns --rm -it --restart=Never -- wget -qO- --timeout=2 <ip>:80

Policy trusts the label, not the pod name/image — the allowed prober just needs the right label. It also needs a working tool: nginx has no wget, so use busybox, and pin :1.28 (latest's wget returns a false connection refused).

Gotchas

  • Separate - = OR; two selectors in one - = AND. Trips everyone.
  • Source ns must carry a label to match — built-in kubernetes.io/metadata.name.
  • CNI without policy support (flannel) = policy silently no-ops.
  • Deny shows as timeout OR connection refused depending on the CNI (drop vs reject). Don't read allow/deny off the error string — judge by the controlled comparison: labeled pod gets in, unlabeled is blocked.
  • Running ≠ serving: a -- sleep override leaves the pod Running but its port closed (→ refused). Verify the target with a probe, not the status phase.
  • Wrong namespace = 0 points. Set it before anything else.

Docs path

  • kubernetes.io search "network policy" → copy the test-network-policy block, trim to the two selectors.
